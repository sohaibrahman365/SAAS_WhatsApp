Platform Overview

GeniSearch is a multi-tenant SaaS platform for WhatsApp-based customer engagement. Each tenant (business) gets their own products, customers, campaigns, and analytics — all managed from one dashboard.

How It Works

Upload products → Import customers → Create a campaign → Send WhatsApp messages → AI analyzes replies → View analytics & insights

Login to the Platform

You can sign in using two methods:

Option A: Email & Password

  1. Enter your email address and password
  2. Click Log In
  3. You'll be redirected to the Dashboard

Option B: Google Sign-In

  1. Click Sign in with Google
  2. Select your Google account
  3. You'll be automatically logged in and redirected
💡
First-time Google users are automatically created with the Analyst role. Ask your Super Admin to upgrade your role if needed.

Create a Tenant (Business)

Each tenant is an independent business with its own products, customers, campaigns, and API keys. Super Admin only.

  1. Go to Settings → Tenants tab
  2. Click + Add Tenant
  3. Fill in the details:
    Business Name
    The company name (required)
    Email
    Primary contact email (required)
    Phone
    Business phone number
    Plan
    Starter ($49) / Growth ($149) / Enterprise ($499)
  4. Click Add Tenant to create
💡
After creating a tenant, go to the API Configuration tab to set up their WhatsApp, META, and AI credentials. Each tenant can have their own API keys, or fall back to the platform master keys.
3

Import Products

You have three ways to add products to a tenant's catalog — no manual entry needed.

Option A: Scrape from Website

Automatically extract products from any e-commerce or business website.

  1. Go to Products page
  2. Click Import → From Website
  3. Paste the website URL (e.g., https://example.com/products)
  4. Click Scan — the platform will extract product names, prices, images, and descriptions
  5. Review the preview table, uncheck any you don't want
  6. Click Import Selected to save

Option B: Import from META Catalog

Pull products directly from a Facebook/Instagram product catalog.

  1. Go to Products page
  2. Click Import → From META
  3. Enter the Catalog ID (or leave blank to use tenant settings)
  4. Enter the Access Token (or leave blank to use tenant settings)
  5. Click Fetch Products
  6. Review and click Import Selected

Option C: Upload Excel / CSV File

Upload a spreadsheet with your product data.

  1. Go to Products page
  2. Click Import → From File
  3. Drag & drop your .xlsx, .xls, or .csv file
  4. The platform auto-maps columns: Name, Price, Description, Category, Image URL
  5. Review the preview and click Import Selected
💡
You can also add products manually using the + Add Product button. Useful for one-off additions.

Add Customers

Customers are the people who will receive your WhatsApp campaigns. Each customer has a phone number (required), plus optional details like name, email, city, and demographics.

Add Customers Manually

  1. Go to Customers page
  2. Click + Add Customer
  3. Fill in the details:
    Phone
    International format, e.g. +971501234567 (required)
    Name
    Customer name
    Email
    Email address
    City / Region
    Location for targeting
    Age / Gender
    Demographics for campaign segments
    Source
    Where they came from (Web, META, TikTok, etc.)
  4. Click Add Customer
⚠️
Phone number format matters. Use international format without spaces: +971501234567. This is the number WhatsApp messages will be sent to.
5

Create & Launch a Campaign

Campaigns send personalized WhatsApp messages to your customers and track their replies.

Create a Campaign

  1. Go to Campaigns page
  2. Click + Create Campaign
  3. Fill in the form:
    Campaign Name
    A descriptive name (e.g., "Summer Sale 2026")
    Product
    Select the product this campaign promotes
    Target Segment
    All Customers, High Priority, New, or Inactive
    Message Template
    Your WhatsApp message with placeholders
    Region / City
    Optional geographic targeting
  4. Click Create — campaign is saved as Draft

Message Placeholders

Use these in your message template — they'll be replaced with real values for each customer:

{{customer_name}}
Customer's name
{{product_name}}
Selected product name
{{discount}}
Discount amount

Example: "Hi {{customer_name}}, check out our new {{product_name}}! Limited time offer."

Launch the Campaign

  1. Find your draft campaign in the table
  2. Click the Launch button
  3. Confirm in the popup — messages are sent immediately
  4. You'll see a summary: Sent: X | Failed: Y | Total: Z
  5. Campaign status changes to Active
💡
After sending, customer replies are automatically analyzed by Claude AI for sentiment (positive/neutral/negative), intent (interested/inquiry/order/feedback), and suggested replies.

CRM — Customer Relationship Manager

The CRM gives you a complete view of every customer with sentiment analysis, behavioral tags, priority scoring, and bulk messaging.

Filter & Search Customers

Use the filter bar to find customers by:

Sentiment
Positive, Neutral, or Negative (based on their replies)
Priority
Low (0-39), Medium (40-69), or High (70-100)
Tags
VIP, At-Risk, Loyal, Churned, etc.
Search
Name, phone, email, city

Tag Customers

Tags help you segment customers for targeted actions.

  1. Select one or more customers using checkboxes
  2. Click Add Tags in the toolbar
  3. Choose preset tags (VIP, At-Risk, New, Inactive, High Spender, Loyal, Price Sensitive, Churned) or type a custom tag
  4. Click Apply Tags

Send Bulk WhatsApp Messages

  1. Select customers using checkboxes (or Select All)
  2. Click Send WhatsApp Message
  3. Type your message (supports {{customer_name}} placeholder)
  4. Click Send — messages go out to all selected customers

Priority Score

Each customer gets an automatic priority score (1-100) based on:

  • Reply rate — how often they respond
  • Sentiment history — positive vs negative responses
  • Conversion rate — purchases or actions taken
  • Recency — when they last engaged

Color-coded: Green (70+) • Yellow (40-69) • Red (<40)

Two-Tier API Architecture

API keys work in a fallback chain:

  1. Tenant-level keys (set in Settings → API Configuration) take priority
  2. If not set, platform master keys (Railway environment variables) are used as fallback
  3. If neither exists, the service runs in stub mode (logs to console, no real API calls)
