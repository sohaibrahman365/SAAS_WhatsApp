## 📊 DATABASE SCHEMA (6 CORE TABLES)

### **Table 1: CAMPAIGNS**
```
id | product_id | name | category | region | country | province | city
target_age_min | target_age_max | target_genders | message_template
sent_count | reply_count | conversion_count | status | created_at
```

### **Table 2: CAMPAIGN_RECIPIENTS (1000 rows for our example)**
```
id | campaign_id | customer_id | customer_name | customer_phone
age | gender | location | message_sent | replied | reply_count
```

### **Table 3: CAMPAIGN_RESPONSES (100 rows - who replied)**
```
id | campaign_id | recipient_id | response_text | response_type
sentiment | sentiment_score | intent | extracted_info
suggested_reply | ai_confidence | received_at
```

### **Table 4: CUSTOMER_ENGAGEMENT_HISTORY**
```
id | product_id | customer_id | total_messages_received | total_messages_replied
reply_rate | conversion_rate | avg_sentiment_score | priority_score (1-100)
positive_responses | negative_responses | last_engagement_date
```

### **Table 5: CUSTOMER_FEEDBACK**
```
id | product_id | customer_id | rating (1-5) | comment
service_quality | product_quality | price_rating | delivery_rating
sentiment | key_themes | tags | created_at
```

### **Table 6: CUSTOMER_PREFERENCES**
```
id | product_id | customer_id | preferred_categories
preferred_price_range | preferred_frequency | best_time_to_contact
do_not_contact | created_at
```
